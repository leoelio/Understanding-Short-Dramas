# Android 原生真机回归验证清单

更新时间：2026-06-10

## 目标

验证 `native-android-migration` 分支的原生 Android App 是否具备稳定演示能力。当前优先验证“半句”原生客户端消费 Web 主线 FastAPI 接口的完整链路，不在手机端运行本地服务端或模型。

## 前置条件

- 分支：`native-android-migration`
- APK：`mobile/banju-native-android/app/build/outputs/apk/debug/app-debug.apk`
- 包名：`com.banju.nativeapp`
- Android SDK：`D:\Android\sdk`
- 电脑服务端：Web 主线 FastAPI，默认端口 `8000`
- 真机开启 USB 调试，`adb devices -l` 能看到 `device`

## 构建与安装

```powershell
.\scripts\build_banju_native_android_debug.ps1
D:\Android\sdk\platform-tools\adb.exe devices -l
D:\Android\sdk\platform-tools\adb.exe install -r mobile\banju-native-android\app\build\outputs\apk\debug\app-debug.apk
```

如果使用 USB 连接电脑服务端：

```powershell
D:\Android\sdk\platform-tools\adb.exe reverse tcp:8000 tcp:8000
```

App 内服务端地址填写：

```text
http://127.0.0.1:8000
```

如果不用 USB reverse，可以填写局域网地址或 Cloudflare HTTPS 隧道地址。

## 启动验证

```powershell
D:\Android\sdk\platform-tools\adb.exe shell cmd package resolve-activity --brief com.banju.nativeapp
D:\Android\sdk\platform-tools\adb.exe shell am start -n com.banju.nativeapp/.MainActivity
```

通过标准：

- App 能启动到登录页或已登录首页。
- 无崩溃弹窗。
- `adb logcat -b crash -d` 没有 `com.banju.nativeapp` 崩溃记录。

## 常见问题定位

### App 连不上服务端

- USB 调试方式：确认已执行 `adb reverse tcp:8000 tcp:8000`。
- 局域网方式：确认手机和电脑在同一网络，服务端监听地址允许局域网访问。
- 公网方式：确认 Cloudflare 隧道仍在线，HTTPS 地址未变化。

### 视频不能播放

- 确认 `/media/episodes/{id}` 在手机可访问。
- 确认服务端返回视频 MIME 类型正确。
- 查看播放页状态文案和 `logcat`。

### 麦克风录音失败

- 确认授予麦克风权限。
- 录音时间不要太短，建议 3-8 秒。
- 朗读固定文本：`同意利用录入声音生成音频`。

### 头像上传失败

- 原图过大时 Android 会裁切压缩为 512x512 JPEG。
- 如果仍失败，检查后端是否返回超过 4MB、格式不支持或鉴权失败。
