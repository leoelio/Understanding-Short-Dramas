# V2 产品底座规划

更新时间：2026-05-28

## 当前正在做

目标：把项目从演示型 MVP 推进到真实产品雏形，先补齐用户身份、权限和登录后选片链路。

本阶段范围：

- 本地账号系统：用户名/密码登录，支持普通用户、复核员、管理员三类角色。
- 登录后选片页：登录后进入短剧列表，支持最近观看入口。
- 权限保护：统计页、复核页、体验配置管理仅管理员和复核员可访问。
- 数据归属准备：互动、弹幕、观看历史开始支持绑定 `user_id`。
- 数据库策略：当前继续 SQLite；表结构按后续 PostgreSQL 迁移设计。

## 近期下一步

- 把后台继续拆成更清晰的管理页面：剧集管理、高光管理、体验配置管理。
- 增加用户管理页：查看用户、角色、活跃状态。
- 增加观看历史和互动数据的用户维度统计。
- 准备公网部署方案，部署前再评估 PostgreSQL 切换。

## 后续版本

### 好友开房和双人同看

依赖当前账号系统。后续增加：

- `rooms`
- `room_members`
- `room_events`
- WebSocket 播放进度同步
- 房间弹幕和互动同步

### AI 二创

第一版不直接做视频生成，先做低风险链路：

- 结尾剧情预测两个选项
- 用户选择后，大模型生成剧情续写
- 输出 3-5 个分镜
- 生成一张海报或漫画图

### 用户带入

先做头像和形象资产管理，再考虑更复杂的换脸或动画形象。

- 用户上传头像
- 生成漫画头像
- 保存到用户资料
- 后续再做“带入模式”

### 客户端和公网访问

Web 先跑通并部署公网。移动端优先考虑：

- 短期：PWA 或 Android WebView 包壳，最快形成手机端可安装体验。
- 中期：React Native 或 Flutter。
- iOS：等 Web 和 Android 体验稳定后再做。

### 语音智能人控制

先做浏览器语音命令 MVP：

- 暂停/继续
- 下一集
- 打开/关闭弹幕
- 调整音量

后续再增加动漫形象和更复杂的智能体对话。
